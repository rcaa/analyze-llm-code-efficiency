Title: Online Majority Element In Subarray
URL: https://leetcode.com/problems/online-majority-element-in-subarray/
Status: OK

solution: 
```
from collections import defaultdict

class MajorityChecker:
    def __init__(self, arr):
        self.element_indices = defaultdict(list)
        for idx, num in enumerate(arr):
            self.element_indices[num].append(idx)
    
    def query(self, left, right, threshold):
        candidates = set()
        subarray_length = right - left + 1
        k = min(20, subarray_length)
        
        # Add first k elements
        for i in range(left, min(left + k, right + 1)):
            candidates.add(arr[i])
        
        # Add last k elements
        for i in range(max(left, right - k + 1), right + 1):
            candidates.add(arr[i])
        
        for num in candidates:
            indices = self.element_indices.get(num, [])
            # Find count in [left, right]
            l = 0
            r = len(indices)
            while l < r:
                mid = (l + r) // 2
                if indices[mid] < left:
                    l = mid + 1
                else:
                    r = mid
            start = l
            r = len(indices)
            while l < r:
                mid = (l + r) // 2
                if indices[mid] <= right:
                    l = mid + 1
                else:
                    r = mid
            end = l
            count = end - start
            if count >= threshold:
                return num
        return -1

arr = [1, 1, 2, 2, 1, 1]
mc = MajorityChecker(arr)
print(mc.query(0,5,4))  # Should return 1
print(mc.query(0,3,3))  # Should return -1
print(mc.query(2,3,2))  # Should return 2
```

efficiency: HIGH

time complexity: O(k log n), where k is the number of candidates (40 in this case) and n is the length of the array.

space complexity: O(n), for storing the indices of each element.

energy implications: LOW

explanation: The solution preprocesses the array to store indices of each element. For each query, it collects candidate elements from the first and last 20 positions of the subarray and checks their counts using binary search. This ensures efficient querying with high performance.
