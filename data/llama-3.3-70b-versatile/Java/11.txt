Title: Maximize Subarray GCD Score
URL: https://leetcode.com/problems/maximize-subarray-gcd-score/
Status: OK

solution:
public class Solution {
    public int maxScore(int[] nums, int k) {
        int n = nums.length;
        int maxScore = 0;
        for (int mask = 0; mask < (1 << n); mask++) {
            if (Integer.bitCount(mask) > k) continue;
            for (int i = 0; i < n; i++) {
                for (int j = i; j < n; j++) {
                    int[] subarray = getSubarray(nums, i, j, mask);
                    int gcd = getGCD(subarray);
                    int score = (j - i + 1) * gcd;
                    maxScore = Math.max(maxScore, score);
                }
            }
        }
        return maxScore;
    }

    private int[] getSubarray(int[] nums, int start, int end, int mask) {
        int[] subarray = new int[end - start + 1];
        int index = 0;
        for (int i = start; i <= end; i++) {
            if ((mask & (1 << i)) != 0) {
                subarray[index++] = nums[i] * 2;
            } else {
                subarray[index++] = nums[i];
            }
        }
        return subarray;
    }

    private int getGCD(int[] nums) {
        int gcd = nums[0];
        for (int i = 1; i < nums.length; i++) {
            gcd = gcd(gcd, nums[i]);
        }
        return gcd;
    }

    private int gcd(int a, int b) {
        if (b == 0) return a;
        return gcd(b, a % b);
    }
}

efficiency: MEDIUM
time complexity: O(2^n * n^2)
space complexity: O(n)

explanation: The solution uses a bitmask to generate all possible combinations of doubling elements in the array, then for each combination, it generates all possible subarrays and calculates their scores by multiplying the length of the subarray with the GCD of its elements. The maximum score is updated accordingly. The time complexity is high due to the generation of all possible combinations and subarrays, but it is necessary to ensure that all possible scenarios are considered.
