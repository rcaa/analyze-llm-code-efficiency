ID: 3569
Title: Maximize Count of Distinct Primes After Split
URL: https://leetcode.com/problems/maximize-count-of-distinct-primes-after-split/
Status: OK

solution: 
```cpp
#include <vector>
#include <unordered_set>

using namespace std;

class Solution {
public:
    vector<int> splitArray(vector<int>& nums, vector<vector<int>>& queries) {
        vector<int> result;
        unordered_set<int> primes;
        for (int i = 0; i < nums.size(); ++i) {
            if (isPrime(nums[i])) {
                primes.insert(nums[i]);
            }
        }
        for (auto& query : queries) {
            int idx = query[0];
            int val = query[1];
            nums[idx] = val;
            int count = 0;
            for (int i = 0; i < nums.size(); ++i) {
                if (isPrime(nums[i])) {
                    count++;
                }
            }
            result.push_back(count);
        }
        return result;
    }
    bool isPrime(int num) {
        if (num <= 1) {
            return false;
        }
        for (int i = 2; i * i <= num; ++i) {
            if (num % i == 0) {
                return false;
            }
        }
        return true;
    }
};
```
efficiency: HIGH
time complexity: O(N*Q*log(N))
space complexity: O(N)
energy implications: LOW 
explanation: The solution iterates through the queries and updates the array. For each query, it checks if the updated element is prime and counts the distinct primes in the array. The `isPrime` function uses a basic primality test with a time complexity of O(sqrt(N)). The overall time complexity is dominated by the nested loops in `splitArray` and `isPrime`, resulting in O(N*Q*log(N)). The space complexity is O(N) due to the `primes` set storing distinct prime numbers. The energy implications are low as the operations are primarily bitwise and arithmetic.
