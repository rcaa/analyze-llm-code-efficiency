ID: 1799
Title: Maximize Score After N Operations
URL: https://leetcode.com/problems/maximize-score-after-n-operations/
Status: OK

solution: 
```cpp
#include <vector>
#include <algorithm>

int maxScore(std::vector<int>& nums) {
    int n = nums.size() / 2;
    std::sort(nums.begin(), nums.end());
    int maxScore = 0;
    for (int i = 0; i < n; i++) {
        int j = nums.size() - 1 - i;
        maxScore += (i + 1) * std::gcd(nums[i], nums[j]);
    }
    return maxScore;
}
```

efficiency: HIGH
time complexity: O(n log n)
space complexity: O(1)
energy implications: LOW
explanation: This solution sorts the array first to group the numbers with the same greatest common divisor together. Then it iterates over the array, choosing the largest numbers at each step to maximize the score. The time complexity is O(n log n) due to the sorting operation, and the space complexity is O(1) as it only uses a constant amount of space. The energy implications are low as the algorithm only performs a few operations.
